iOS中的单例

发表于 2015-06-03   |   分类于 iOS心经   |  

什么是单例

单例是是程序开发中一种常见的设计模式。保证在程序运行过程，一个类只有一个实例。

在面向对象语言中，我们使用一个类获取对象时，如果“多次创建对象”或者“多次获取对象”，得到的都是同一个对象，那么这个对象就是单例对象——同时只存在一个实例对象。

创建对象时使用的类被称为单例类。

单例对象的特点

单例对象具有如下特点：

  1. 可以被全局访问
  2. 不会被释放
  3. 创建过程只执行一次

单例对象应用场景

单例对象通常应用于当一个对象需要在程序中的任意地方调用的时候。

  1. 某个类经常使用（节约系统资源）
  2. 定义工具类
  3. 共享数据(界面中传值，但是不常用)

例如：

  • 在设备旋转时，使用UIApplication.SharedApplication获取程序，进一步获取设备当前旋转方向。

  • 进行数据存储时，[NSUserDefaults standardUserDefaults] 来设置用户偏好。

  • 使用数据库连接类对数据库进行操作时，需要保证操作的是同一个数据库。

单例使用的注意点

不要继承单例类。如果继承成了，会出现下面的问题：（因为一次性代码在程序中只执行一次）

  • 先创建子类永远是子类对象
  • 先创建父类永远是父类对象

用Objective-C语言实现单例

编写完整的单例要求比较高，不仅要求通过方法获取的对象是单例，如果对该对象进行copy mutableCopy copyWithZone 等操作时也要是同一份对象。所以要求我们必须重写这些方法。

一般情况下，要实现单例，都会给单例提供一个快速创建实例的类方法

###单例的类方法的命名规则
以share或者default开头，后面跟上类名。比如 + (instancetype)shareTools

在ARC下实现

思路主要是这个单例类在第一次alloc的时候分配一次内存，以后再alloc的时候直接返回第一次分配的那个内存而不是重新分配。这里采用 dispatch_once 方法来达成目的，这样可以确保线程安全。

另外，为了确保 copy 和 mutableCopy 等操作时不进行深拷贝，返回的也是同一个对象，所以要重写 copyWithZone 和 mutableCopyWithZone 方法。（需要实现NSCopying和NSMutableCopying协议）

+ (instancetype)shareTools 
{
    return [[self alloc] init];
}


// alloc的时候，会调用allocWithZone方法，给对象分配内存
+ (instancetype)allocWithZone:(struct _NSZone *)zone
{
    static id instance;

    /* 不推荐这种写法，有可能线程不安全
    if (instance == nil) {
        instance = [super allocWithZone:zone];
    }
     */

    static dispatch_once_t onceToken;

    dispatch_once(&onceToken, ^{

        instance = [super allocWithZone:zone]

    });

    return instance;
}


- (id)copyWithZone:(NSZone *)zone
{
    return self;
}


- (id)mutableCopyWithZone:(NSZone *)zone
{
    return self;
}

在MRC下实现

在MRC下，单例的主要创建方法和ARC都一致，但是需要额外处理 release，retain 和 retainCount 等方法，这样这个单例对象既不会被销毁，也不会增加计数。

  • release方法中什么也不做
  • retain方法直接返回自己，不进行其他操作

  • retainCount方法中返回一个很大的数或者是一个负数，这样有经验的程序员就会知道这是一个单例

    + (instancetype)shareTools 
{
return [[self alloc] init];
}

+ (instancetype)allocWithZone:(struct _NSZone *)zone
{
static id instance;

static dispatch_once_t onceToken;

dispatch_once(&onceToken, ^{
    instance = [super allocWithZone:zone]
});

return instance;
}


- (id)copyWithZone:(NSZone *)zone
{
    return self;
}


- (id)mutableCopyWithZone:(NSZone *)zone
{
    return self;
}


- (oneway void)release
{
    // 这个单例对象的release方法什么都不做
}


- (instancetype)retain
{
    return self;
}


- (NSUInteger)retainCount
{
    return MAXFLOAT;
}

把单例抽取成宏

一个项目中可能存在多个单例，如果每一个单例都要重新实现一次，这样会造成大量的代码冗余。又因为单例无法继承，所以最好方式是把实现单例的代码抽取成一个带参数的宏，可以给不同单例使用。

  • 声明

    #define SingleInterface(name) +(instancetype)share##name

  • 实现

    #if __has_feature(objc_arc)
// ARC
#define SingleImplenmentation(name) +(instancetype)share##name \
{\
return [[self alloc] init];\
}\
+ (instancetype)allocWithZone:(struct _NSZone *)zone\
{\
    static id instance;\
    static dispatch_once_t onceToken;\
    dispatch_once(&onceToken, ^{\
    instance = [super allocWithZone:zone]\
});\
    return instance;\
}\
- (id)copyWithZone:(NSZone *)zone\
{\
    return self;\
}\
- (id)mutableCopyWithZone:(NSZone *)zone\
{\
    return self;\
}\
#else
// MRC
#define SingleImplenmentation(name) +(instancetype)share##name \
{\
return [[self alloc] init];\
}\
+ (instancetype)allocWithZone:(struct _NSZone *)zone\
{\
    static id instance;\
    static dispatch_once_t onceToken;\
    dispatch_once(&onceToken, ^{\
    instance = [super allocWithZone:zone]\
});\
    return instance;\
}\
- (id)copyWithZone:(NSZone *)zone\
{\
    return self;\
}\
- (id)mutableCopyWithZone:(NSZone *)zone\
{\
    return self;\
}\
- (oneway void)release\
{ }\
- (instancetype)retain\
{\
    return self;\
}\
- (NSUInteger)retainCount\
{\
    return MAXFLOAT;\
}\
#endif

    上面代码中，判断是ARC还是MRC的宏是：

    #if __has_feature(objc_arc)
//ARC
#else
//MRC
#endif

抽取成宏之后，代码的阅读性很差，不过这些代码只需要痛苦一次，以后实现单例把上述代码直接拷贝进来即可。大致使用步骤如下：

  • 把上面的宏声明和宏实现的代码写到一个Single.h文件中
  • 某个类(比如 NetworkTools)想成为单例，#import "Single.h"
  • 在NetworkTools.h中写上SingleInterface(NetworkTools)
  • 在NetworkTools.m中写上SingleImplementation(NetworkTools)

OC下另一个单例的写法（简洁快速）

OC中，简洁快速写一个可以用的工具类单例，以 NetworkTools为例

+ (instancetype)shareNetworkTools
{
    static NetworkTools *instance;
    static dispatch_once_t onceToken;
    dispatch_once(&onceToken, ^{
        instance = [[NetworkTools alloc] init];
    });
    return instance;
}

用Swift语言实现单例

在Swift语言中实现单例比OC中简单，并且有多种实现方式(仍以NetworkTools为例)：

  • 第一种方式：

    static var instance: NetworkTools?
static var onceToken: dispatch_once_t = 0

class func shareNetworkTools() -> NetworkTools
{
    dispatch_once(&onceToken) { () -> Void in
        instance = NetworkTools()
    }
    return instance!
}

  • 第二种方式：

    // Swift中的let是线程安全的，let只能赋值一次
static let instance: NetworkTools = NetworkTools()

override init() {

}

class func shareNetworkTools() -> NetworkTools
{
    return instance
}

  • 第三种方式：(最推荐这样编写)

    // Swift中单例的命名规范：以share开头，后面跟上Instance
static let shareInstance = NetworkTools()
