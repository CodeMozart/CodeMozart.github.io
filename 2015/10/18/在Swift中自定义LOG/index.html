在Swift中自定义LOG

发表于 2015-10-18   |   分类于 iOS心经   |  

前言

在我们开发程序过程中，为了调试各种问题，通常需要print出很多内容。然而，首先这些打印会耗费不少的程序性能，其次这些打印对用户来说是毫无意义的。

那我们怎么解决这个问题呢？你可能会说，这个好办，只要在程序发布之前手动把所有的print都去掉或注释掉就可以了。这确实是一个办法，但是对于中大型项目来说，可能有成百上千个打印在里面，如果一个一个去掉无疑太耗费时间。作为一个程序员，怎么能容忍这么低效做法，所以我们希望通过自定义LOG的方法来完美的解决这个问题。

自定义LOG有两个目的：

  • 在程序开发阶段自动显示LOG
  • 在程序发布阶段自动屏蔽LOG

下面就介绍一下如何在Swift项目中自定义LOG.

在Swift项目里自定义LOG

1.自定义的LOG写在哪里

Swift项目很有特点，在同一个命名空间下面，全局都可以看作是Main函数。所以自定义的LOG函数写在哪里都可以被其他文件调用。为了避免误会，这里把自定义的LOG函数写在AppDelegate.swift文件下面。

2.自定义LOG的参数

从上面的图片可以看到，这里自定义的LOG的名称是

func MHzLog<T>(message: T, method: String = __FUNCTION__, line: Int = __LINE__)

解释一下里面的参数：

  • message : T : T是高级语法，泛型。可以理解为传什么就是什么。传的是Int类型，打印出来就是Int值；传的是String类型，打印出来就是String.

  • method: String = __FUNCTION__ 这里给形参method设置了默认值__FUNCTION__，这样在调用的时候不需要给参数method传值。

    print(__FUNCTION__)是打印当前内容所在的方法，自定义LOG里面也是如此。

  • line: Int = __LINE__ 这里给形参line设置了默认值__LINE__，这样在调用的时候不需要给参数line传值。

    print(__LINE__)是打印当前内容所在的行，自定义LOG里面也是如此。

3.自定义LOG打印的内容

从上一步我们知道，因为参数的设置，让自定义的LOG不仅可以打印想要打印的内容本身，同时也可以打印出其所在的方法，以及所在的行。所以自定义LOG的具体实现如下

print("\(method)[\(line)]: \(message)")

拼接完，打印显示的格式为：方法名［所在行］：打印内容

4.实现最终目的

之前我们说过，自定义LOG的最终目的是：

  • 在程序开发阶段自动显示LOG
  • 在程序发布阶段自动屏蔽LOG

下面我们就来实现它。

4.1 添加DEBUG标记

首先需要知道的是：Swift中没有宏定义。所以利用宏来解决这个问题的道路已经堵死了，不过别灰心，，我们可以自定义通过添加标记来解决这个问题。

下图是给Swift项目添加DEBUG标记的步骤，按图操作即可：

4.2 实现代码

在添加好标记之后，我们只需要在自定义的LOG函数里面写上几行简单的代码即可全部搞定。

#if DEBUG
print("\(method)[\(line)]: \(message)")
#endif

这样，在DEBUG的情况下，自定义LOG会打印内容，而在Release下，什么也不会打印。
